## 1. 배경 및 목적

### 리팩토링이 필요한 이유

현재 코드와 운영 흐름에서는 UTC 시각 문자열을 여러 방식으로 다루고 있다.
특히 워크플로우 작업 생성 시각이나 테스트 기준 시각 같은 값은 일관된 포맷이 중요하지만,
쉘의 `date -u +"%Y-%m-%dT%H:%M:%SZ"` 같은 외부 명령 의존 또는 개별 모듈의 포맷 로직에 기대면
재사용성과 테스트 가능성이 떨어진다.

이 작업의 목적은 UTC 시각 포맷을 코드 레벨 유틸로 통합해,
CLI 내부 로직과 테스트가 동일한 기준 함수를 사용하도록 만드는 것이다.

### 기대 효과

- [x] UTC 날짜/시간 포맷 규칙이 한곳에 모인다
- [x] 쉘 명령 의존 없이 코드에서 같은 포맷을 재사용할 수 있다
- [x] 테스트에서 deterministic한 날짜 포맷 검증이 쉬워진다
- [x] 향후 `spec approve`, `plan approve`, `done` 같은 상태 전이 명령에서 같은 유틸을 재사용할 수 있다

## 2. 현재 상태 (As-Is)

### 문제 코드 / 구조

```
파일: src/core/start.ts
파일: 테스트/운영 흐름에서 사용하는 UTC date shell command
```

현재는 `formatUtcTimestamp()`가 `start` 쪽에만 묶여 있고,
운영 중에는 별도로 `date -u` 쉘 명령을 쓰는 경우가 있다.

### 측정 지표 (개선 전)

| 지표               | 현재 값           | 목표 값       |
| ------------------ | ----------------- | ------------- |
| UTC 포맷 함수 위치 | 개별 모듈/쉘 분산 | 공용 유틸 1곳 |
| 날짜 포맷 재사용성 | 낮음              | 높음          |
| 테스트 용이성      | 보통              | 높음          |
