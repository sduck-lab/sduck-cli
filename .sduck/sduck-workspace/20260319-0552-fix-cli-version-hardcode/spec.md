## 1. 버그 개요

### 문제 요약

`src/cli.ts`에서 Commander 버전 설정이 `.version('0.1.0')`로 하드코딩되어 있어 `package.json` 버전과 CLI 출력 버전이 쉽게 불일치할 수 있다.

## 2. 재현 방법 (Steps to Reproduce)

**환경 정보**
| 항목 | 내용 |
|------|------|
| 환경 (local / dev / staging / production) | local |
| OS / 브라우저 / 앱 버전 | macOS CLI 환경 |
| 계정 유형 / 권한 | 해당 없음 |
| 관련 데이터 ID / 조건 | `package.json` 버전과 `src/cli.ts` 하드코딩 값이 달라진 상태 |

**재현 단계**

1. `package.json`의 `version` 값을 변경한다.
2. `src/cli.ts`의 `.version('0.1.0')` 하드코딩은 그대로 둔다.
3. `sduck --version`을 실행한다.

**실제 결과 (Actual)**

CLI가 패키지 메타데이터가 아닌 하드코딩된 버전을 출력해 실제 릴리즈 버전과 어긋날 수 있다.

**기대 결과 (Expected)**

CLI가 단일 소스의 버전 메타데이터를 사용해 항상 현재 패키지 버전과 일치하는 값을 출력해야 한다.

## 3. 원인 분석 (Root Cause)

### 원인 파악 현황

- [ ] 원인 특정 완료
- [x] 원인 조사 중
- [ ] 원인 불명

### 원인 요약

버전 문자열이 `package.json`과 별도로 `src/cli.ts`에 중복 정의되어 있다. 이 구조에서는 버전 업데이트 시 두 위치를 함께 수정해야 하므로 누락 위험이 있다.

### 원인 코드 위치

```text
파일: src/cli.ts, src/core/command-metadata.ts, package.json
함수 / 라인: program.version(...) 설정부와 관련 메타데이터 정의부
```

### 발생 조건

버전 릴리즈나 수동 버전 변경 시 `package.json`만 수정하고 CLI 소스의 하드코딩 문자열을 놓치면 발생한다.

## 4. 수정 방안

### 방안 검토

| 방안   | 설명                                                         | 장점           | 단점                           |

| 방안 A | `src/cli.ts` 하드코딩만 최신 값으로 수동 갱신                | 빠름           | 같은 문제가 반복됨             |
| 방안 B | CLI 버전을 공용 메타데이터 또는 패키지 정보 단일 소스로 연결 | 재발 방지 가능 | 메타데이터 로딩 방식 검토 필요 |

### 선택한 방안 및 이유

> **선택:** 방안 B
>
> **이유:** 단순 문자열 교체로는 이후 릴리즈에서 같은 불일치가 반복된다. 버전 정보를 단일 소스로 연결해야 유지보수 비용과 실수 가능성을 줄일 수 있다.

### 변경 파일 목록 (예상)
