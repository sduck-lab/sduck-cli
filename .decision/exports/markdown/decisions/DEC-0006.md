# DEC-0006: Bind confirmation to a canonical digest inside the lifecycle transaction

## Decision
Confirm only when task ID, deterministic brief digest, revision, and structured provenance match source regenerated under the workspace lock.

## Rationale
- Client permission dialogs and MCP annotations cannot prove human review.
- Separate preview and confirmation reads allow stale-brief confirmation.
