# DEC-0013: Separate context reads from refresh mutations

## Decision
get_context is read-only; refresh_context is a separately named idempotent mutation. Task creation does not include expensive indexing.

## Rationale
- Read-only MCP annotations must match behavior.
- Scanning under writer lock creates partial-operation and contention risks.
